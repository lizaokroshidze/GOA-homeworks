1. Git და GitHub: - Git: ვერსიების კონტროლის სისტემა, რომელიც შენს კომპიუტერზე მუშაობს. საშუალებას გაძლევს track-ს გაუწიო ცვლილებებს, დაბრუნდე ძველ ვერსიებზე, შექმნა branch-ები, merge-ები და ა.შ. - GitHub: ონლაინ პლატფორმა Git რეპოზიტორიებისთვის. საშუალებას გაძლევს ატვირთო რეპო, თანამშრომლობდე სხვა ადამიანებთან, შექმნა pull request-ები, issues, Wiki და ა.შ. - განსხვავება: Git არის ადგილობრივი ინსტრუმენტი, GitHub არის ინტერნეტი/სერვისი. 2.Commit: - Commit არის ცვლილებების snapshot. - გვჭირდება, რომ შეგვეძლოს დაბრუნება ძველ ვერსიებზე, ცვლილებების ისტორიის შენახვა და თანამშრომლობა სხვა დეველოპერებთან. 3. Main branch და სხვა branch-ები: - Main branch: ძირითადი, სტაბილური ვერსია პროექტისთვის. - სხვა branch-ები: ცალკე სივრცეები ახალი ფუნქციონალისთვის ან ექსპერიმენტებისთვის. - Branch-ების გამოყენება: ცვლილებების იზოლირება, teamwork, სტაბილური main-ის შენარჩუნება.
